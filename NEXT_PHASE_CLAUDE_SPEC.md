# 小瀾秘書下一階段升級規格書

這份文件是要交給 Claude 實作的開發規格。請先完整閱讀，再改程式。

## 目前狀態

專案是 LINE Bot「小瀾」，部署在 Vercel，主要檔案如下：

- `api/webhook.js`：LINE webhook、Claude 對話、tool use、記帳、待辦、員工回報。
- `api/reminder.js`：定時提醒、早晚摘要、待辦與帳務盤點。
- `setup-db.sql`：Supabase 資料表。
- `index.html`：前端管理頁。
- `STAFF_REPORT_AGENT.md`：員工回報功能說明。

目前小瀾已經有：

- 私訊可以自然對話，不需要 `#`。
- 群組只處理 `#` 或 `＃` 開頭的訊息，避免誤觸發。
- `#回報` 是員工缺貨、破損、少件回報。
- 可以辨識員工拍的運單照片，OCR 後寫入 Google Sheet。
- 可以記待辦、完成待辦、刪除待辦、延後待辦。
- 可以標記待辦狀態：待處理、進行中、半完成、等待回覆、未完成。
- 可以記帳，並用卡片修改公司/私人、分類、刪除。
- 可以儲存網址與筆記，並在提問時查回來。
- 提醒時已經會帶待辦狀態與今日帳務摘要。

## 這一階段的核心目標

把小瀾從「會記錄的 LINE Bot」升級成「比較像真人秘書的 Agent」。

重點不是多做很多指令，而是讓它做到：

1. 會判斷現在是在私訊還是群組。
2. 私訊中可以自然對話，不需要死板指令。
3. 群組中只有 `#` 開頭才回覆或紀錄。
4. 待辦、記帳、提醒都要有可點選卡片，少叫使用者打字。
5. 已完成、半完成、等待回覆、未完成都可以被小瀾追蹤。
6. 提醒時不是只列清單，而是會問下一步。
7. 小瀾要能主動整理「今天該先處理什麼」。

## 絕對不能破壞的規則

### 私訊與群組規則

這條最重要。

私訊：

- 使用者跟小瀾一對一聊天時，不需要任何關鍵字。
- 可以自然問問題、交辦事情、記帳、傳圖片、查待辦。
- 不可以因為訊息沒有 `#` 就忽略。

群組：

- 只有訊息以 `#` 或 `＃` 開頭才可以處理。
- 沒有 `#` 的群組訊息，一律不要回覆、不要記錄、不要判斷待辦。
- `#回報` 保留給員工缺貨/破損回報流程。
- `#待辦` 或其他 `#內容` 才走一般秘書 Claude 流程。

### 回覆風格

小瀾不要像工程師，也不要像客服罐頭。

風格：

- 繁體中文。
- 簡短。
- 像秘書，不要說教。
- 不要一直叫「主管稍後確認」，要說「小瀾稍後確認」。
- 不要亂猜，如果不確定要問一句關鍵問題。
- 不要把 webhook 網址當成新系統網址。
- 不要在 URL 後面多加 `**` 或多餘 Markdown。

### 待辦判斷

不要把每一句話都記成待辦。

待辦是：

- 有明確要執行的動作。
- 例如：買、叫貨、付款、確認、回覆、安排、通知、處理、追蹤、上架、製圖。

筆記是：

- 資訊、網址、規則、帳號、廠商資料。
- 例如：「薪資系統網址是 xxx」應該存成筆記，不是待辦。

聊天是：

- 問問題、抱怨、閒聊、測試。
- 不要存待辦。

## 本階段要完成的功能

## 功能 1：提醒推播改成卡片

目前 `api/reminder.js` 的提醒多數是純文字。

請改成：

- 待辦提醒要附 LINE Flex Message 卡片。
- 每一項待辦要有按鈕：
  - 完成
  - 半完成
  - 等待
  - 延後明天
- 如果是帳務提醒，要附按鈕：
  - 今天帳務
  - 本月帳務
  - 最近一筆算公司
  - 最近一筆分類

### 實作方向

目前 `api/webhook.js` 已經有：

- `buildTodoActionFlex(todos, stateMap)`
- `buildExpenseCategoryFlex(expenseId)`
- `buildExpenseFlexMessage(...)`
- `replyMessage(...)`

但 `api/reminder.js` 是 push message，不一定能直接共用 `replyMessage`。

請在 `api/reminder.js` 補：

- `pushMessages(to, messages)`，支援 text + flex array。
- `buildReminderTodoFlex(todos, stateMap)`。
- `buildExpenseReminderFlex()`。

不要把 `api/webhook.js` 整份 require 進來，避免 serverless 互相污染。可以複製必要的 Flex builder，或整理成 `api/lib/line-flex.js`。

如果要整理共用檔，建議：

- 新增 `api/lib/line.js`
- 新增 `api/lib/flex.js`

但請保持改動不要太大。

### 驗收

下午提醒與晚上總結時：

- LINE 會收到文字摘要。
- 同時收到可以點的待辦卡片。
- 點「完成」會送出 `完成第1項`。
- 點「半完成」會送出 `半完成第1項`。
- 點「等待」會送出 `等待第1項`。
- 點「延後明天」會送出 `延後第1項到明天`。

## 功能 2：私訊「盤點」變成秘書式整理

現在使用者輸入 `盤點` 或 `待辦`，小瀾會列待辦。

請升級成：

使用者在私訊輸入：

- `盤點`
- `今天要做什麼`
- `今天先做什麼`
- `幫我整理一下`
- `我現在該做什麼`

小瀾要回：

1. 今日最重要 3 件事。
2. 等待回覆的事。
3. 半完成的事。
4. 今天帳務狀況。
5. 可以直接點的待辦卡片。

### 排序邏輯

優先順序：

1. priority = urgent。
2. 狀態是未完成。
3. 狀態是半完成。
4. 狀態是等待回覆。
5. due_date 是今天或已過期。
6. 建立時間較早。

### 回覆範例

```text
今天我會先看這幾件：

1. 招牌尺寸確認
2. 1688 廠商折扣回覆
3. 薪資系統上線確認

等待回覆：
- 88折平台報價

半完成：
- 舒肥雞做圖文

今日帳務：
支出 NT$1,250，公司 NT$800，私人 NT$450。

你可以直接點下面卡片處理。
```

## 功能 3：待辦狀態追問

小瀾要能在提醒時追問狀態，不只是列清單。

例如有待辦超過 2 天未完成：

```text
這 2 件卡住比較久了，要我幫你標成等待回覆，還是延後到明天？
```

卡片按鈕：

- 等待回覆
- 延後明天
- 刪除

### 實作方向

可先用簡單規則，不用太複雜：

- `created_at` 超過 2 天，且狀態是 `待處理` 或 `未完成`，就列入「卡住」。
- `created_at` 超過 5 天，提醒文字加上 `這件已經放比較久了`。

## 功能 4：更準確的完成/取消/延後理解

目前已經有 `complete_todo`、`delete_todo`、`postpone_todo` tool。

請加強：

使用者自然說：

- `薪資系統好了`
- `新系統網址不用記了`
- `舒肥雞文案先等老闆回`
- `招牌圖做到一半`
- `1688 那個明天再處理`

Claude 要能對應到現有待辦或筆記。

### 實作方向

在 `SYSTEM_PROMPT` 加強規則：

- 如果使用者說「好了、完成、弄好了、處理好了」，要找最相近待辦並呼叫 `complete_todo`。
- 如果使用者說「先等、等回覆、等老闆、等廠商」，要標記 `等待回覆`。
- 如果使用者說「做到一半、還沒完全好、半完成」，要標記 `半完成`。
- 如果使用者說「不用、取消、不要記、刪掉」，要刪除待辦或筆記。

目前 `markTodoStatus(n, status)` 只能用第 N 項。

請新增 keyword 版工具：

- `mark_todo_status`

輸入：

```json
{
  "keyword": "薪資系統",
  "status": "等待回覆"
}
```

支援狀態：

- `進行中`
- `半完成`
- `等待回覆`
- `未完成`

## 功能 7：員工回報流程更穩

目前 `#回報` 可以用，但要避免在群組誤判。

請確認：

- 群組中只有 `#回報` 才進員工回報流程。
- 其他圖片不可以觸發 OCR。
- `#回報 少3 白色少2` 如果沒有照片，要回「請附運單照片」。
- 如果先傳照片再打 `#回報 少3`，可以使用最近一張照片。
- 如果 OCR 找不到運單號，要請員工補打運單號。

回覆文字請用：

```text
小瀾已建立回報，但找不到運單號。
運單號：xxxx
請主管稍後確認。
```

其中最後一句請改成：

```text
小瀾稍後確認。
```

## 成功標準

這一階段完成後，小瀾應該讓使用者感覺：

- 私訊可以正常聊天，不像關鍵字機器人。
- 群組不會亂插話。
- 待辦不是只會記，還會追蹤狀態。
- 提醒不是只叫人做事，而是能整理重點。
- 記帳可以快速修正，不用一直打完整指令。
- 小瀾知道哪些事完成、哪些卡住、哪些在等回覆。

## 不要做的事

- 不要改 LINE channel token、Google token、Supabase key。
- 不要重設資料庫。
- 不要刪既有資料。
- 不要把所有功能拆成大型重構，先完成可用功能。
- 不要讓群組沒 `#` 的訊息進 Claude，會浪費 token 且容易誤記。
- 不要把所有問題都硬寫 if/else，能交給 Claude tool use 的就交給 Claude。

## 最後提醒

使用者要的是「秘書」，不是「更多指令」。

所以每個功能請優先思考：

- 能不能少打字？
- 能不能用卡片點？
- 能不能自動判斷狀態？
- 能不能在不確定時只問一句？
- 能不能避免在群組亂回？

做到這些，小瀾才會開始像真正的 AI Agent。
