## 2. 🔑 環境變數現況（Vercel）＋一個大雷

### ⚠️ 大雷：Vercel「Sensitive」變數，部署後的程式讀不到！
這個專案踩過的最大坑：標成 **「Sensitive」**（環境設「Production and Preview」、Development 被鎖）的環境變數，**serverless function 在執行時讀不到**。只有標 **「All Environments」的非 Sensitive 變數**讀得到。而且 **Sensitive 一旦存檔不能改回普通**，只能刪掉重建。

| 讀得到 ✅（All Environments） | 讀不到 ❌（Sensitive） |
|------|------|
| LINE_CHANNEL_SECRET / ACCESS_TOKEN | STAFF_REPORT_SPREADSHEET_ID |
| SUPABASE_URL / ANON_KEY | STAFF_REPORT_IMAGE_FOLDER_ID |
| ANTHROPIC_API_KEY | GOOGLE_VISION_API_KEY |
| GOOGLE_CLIENT_ID / SECRET / REDIRECT_URI | STAFF_REPORT_SHEET_NAME / ORDER_SHEET_NAME |
| GOOGLE_REFRESH_TOKEN | STAFF_LIFF_ID |

### 怎麼繞過（已實作，不用再處理）
- **Sheet ID／資料夾 ID／LIFF ID**：非機密，直接**寫死後備值**在 `api/staff-report.js`、`staff.html`、`webhook.js`（env 有值才優先用 env）。
- **分頁名稱**：程式有預設值（`員工問題回報`／`所有訂單`），沒讀到也沒差。
- **Google refresh token**：改成**優先讀 Supabase**（見 §5），env 當後備。
- **Vision 金鑰**：是真金鑰、不能寫死 → 所以「拍運單照片自動讀號」目前不會動（用掃碼/手打運單號就好）。

> 💡 之後若要徹底乾淨：把上面那些 Sensitive 變數**刪掉、用普通（非 Sensitive、All Environments）重新建立**，程式就讀得到了。

## 3. 📋 總倉回報怎麼運作（這次的主角）

> **用詞規定**：UI 一律「**總倉**／**小瀾**」，不要「員工／主管」。但 Sheet 分頁名稱仍是 `員工問題回報`、欄位標題仍含「員工」＝資料結構，**別改否則會寫到別的分頁**。

1. 群組打 `#回報` → 卡片出現 **「📋 開回報表單」** → 開 `staff.html`
2. 第一頁問 **「貨到了嗎？」**
   - **🚫 未到貨**：只填運單號（可多筆）→ 送出
   - **✅ 有到貨**：步驟2 運單（掃碼／拍運單照片／手打）→ 步驟3 問題照片（多張）→ 自由描述框（例「粉色少3、白色少2」）→ 送出
3. 送出 → `POST /api/staff-report` → 每個運單號**各寫一列**進 Google Sheet，描述原話進「員工文字」欄、照片進各自欄位
4. 聊天版 `#回報`（少3／破2／未到貨／口語 AI 補刀）仍保留當備援

## 5. 🔄 Google 授權失效怎麼救（重要！）

**症狀**：表單送出回 `invalid_grant`，或寫 Sheet/Drive/Calendar 失敗。
（注意：問小瀾「這週行程」是讀 Supabase **快取** `xlan_events`，看起來正常**不代表** token 還活著！）

**修法（不用碰 Vercel、即時生效）**：
1. 開 `https://xlan-secretary-rqhb.vercel.app/api/oauth`
2. 按「用 Google 登入授權」→ 用**香奈的 Google 帳號**登入、同意
3. 看到綠字「✅ 已自動存進系統」就好 → 新 token 自動存進 **Supabase `xlan_kv` 的 `google_refresh_token`**，`staff-report.js` 即時讀取

> `api/oauth.js` 取代了原本不見了的 `get-token.js`。
> ⚠️ 注意：`webhook.js` 的 Google client **還是只讀 env 的 GOOGLE_REFRESH_TOKEN**（沒走 Supabase），所以聊天版功能若也要寫 Google，得另外把 env 的 token 也更新、或把 webhook 也改成讀 Supabase。

## 7. 🤖 給下一個 AI 的提醒

1. **`webhook.js` 是 Claude 的地盤**：211KB 的大檔、正式環境在用，改之前看清楚，小步改、每改必 `node --check`。
2. **不確定就先問香奈**：對不上、沒把握、要刪資料/改文件時，先問，別亂動。
3. **push 前先 `gh auth status` 確認是 www161616**，否則 403。
4. **改 bot 功能要同步更新文件**：CLAUDE.md、XLAN_SECRETARY_USAGE_AND_TESTS.md、（總倉相關）STAFF_REPORT_AGENT.md。
5. **遇到「設定對了卻沒生效」先想 §2 那個 Sensitive 雷**，別假設「webhook 讀得到＝表單也讀得到」（兩個 function 一致地讀不到 Sensitive 變數）。
6. **診斷招式**：臨時在 GET handler 回 `!!process.env.X` 的布林（不洩漏值）+ 用 `getAccessToken()` 測 Google，curl `?diag=1` 看；用完**要移除**。
7. **非機密 ID（Sheet/資料夾/LIFF）可以寫死後備值**；**真金鑰（Vision/Anthropic/LINE token）絕不寫進程式碼**。

   - LIFF init 讀不到 ID（`liff.state` 包住網址參數）→ 改成跟後端拿 + 寫死後備。
